# ThinQ Mom 시연 대본

발표자가 심사위원 앞에서 그대로 따라 할 수 있는 데모 대본입니다.  
브랜드명은 **ThinQ Mom**으로 통일합니다.

**권장 화면 순서:** `/hub` → (모드 3회) → `/wife` → `/husband`  
**권장 기기:** 모바일 375px 또는 동일 비율 브라우저

---

## 1. 도입

> “ThinQ Mom은 임산부가 직접 복잡한 설정을 하지 않아도, 평소처럼 말한 한마디를 AI가 해석해서 집 안 환경을 바꿔주는 케어 서비스입니다.  
> 기존 ThinQ가 가전을 움직이는 서비스였다면, ThinQ Mom은 임산부가 힘든 순간의 **조건**을 바꾸는 서비스입니다.”

**보조 설명 (필요 시):**

- ThinQ ON 허브를 중심으로, 음성·텍스트 한 문장으로 케어 모드가 실행됩니다.
- 아내 화면에는 오늘 필요한 케어가, 남편 화면에는 **행동 중심** 제안이 전달됩니다.
- 공기청정기는 실제 ThinQ API로 제어하고, 3D 시뮬레이션·화면 업데이트와 연동됩니다.

---

## 2. Hub 첫 화면 소개

**화면:** `/hub` 접속 — 중앙에 **ThinQ ON 이미지 하나만** 표시

> “먼저 Hub 화면입니다.  
> 처음 화면에서는 별도의 버튼이나 설명 없이 **ThinQ ON 허브**만 보입니다.  
> 복잡한 앱 메뉴를 먼저 보여주는 것이 아니라, 집 안 허브 하나로 시작하는 경험입니다.”

**동작:** ThinQ ON 이미지를 터치해 패널을 엽니다.

> “사용자가 허브를 터치하면, 음성 또는 텍스트로 자신의 상태를 말할 수 있습니다.  
> 시연장에서는 음성이 불안정할 수 있어, 동일한 흐름을 **예시 문장**이나 **텍스트 입력**으로도 실행할 수 있게 준비했습니다.”

---

## 3. 입덧모드 시연

**사용 발화 (메인):**

> “나 지금 입덧이 너무 심해”

**예비 발화 (칩 또는 텍스트):**

- “음식 냄새 때문에 아무것도 못 먹겠어”
- “속이 계속 울렁거려”
- “밥 냄새가 역겨워”

**조작:** Hub 패널 → 시연용 발화 예시 **입덧모드** 칩 클릭 (또는 텍스트 입력 후 실행)

**설명 멘트:**

> “예를 들어 ‘나 지금 입덧이 너무 심해’라고 말하면, ThinQ Mom은 이 문장을 단순 명령이 아니라 **냄새 부담**과 **식사 어려움**의 신호로 해석합니다.  
> 그래서 **공기청정기**를 작동시키고, 집 안 환경을 냄새 부담을 줄이는 방향으로 전환합니다.  
> 3D 시뮬레이션은 **입덧 케어 장면(NAUSEA_SCENE)** 으로 전환되고,  
> 동시에 **아내 화면**에는 오늘 필요한 케어가 정리되고, **남편 화면**에는 ‘오늘 저녁은 향이 강하지 않은 담백한 메뉴가 좋아요’처럼 **행동 중심** 제안만 전달됩니다.”

**확인 포인트 (짧게 언급 가능):**

- 감지 모드: **입덧모드** (NAUSEA_MODE)
- 공기청정기 actual action 결과
- localStorage `thinq-mom-demo-scene` = `NAUSEA_SCENE`

---

## 4. 수면모드 시연

**사용 발화 (메인):**

> “나 이제 잘 준비 할래”

**예비 발화:**

- “요즘 잠을 제대로 못 자”
- “너무 피곤해서 눕고 싶어”
- “몸이 천근만근이야”

**조작:** Hub 패널 → **수면모드** 칩 클릭

**설명 멘트:**

> “이번에는 ‘나 이제 잘 준비 할래’라고 말해보겠습니다.  
> ThinQ Mom은 사용자가 직접 ‘수면모드 켜줘’라고 말하지 않아도, **피로**와 **취침 의도**를 이해해서 수면모드로 연결합니다.  
> 공기청정기는 **수면에 맞는 상태**로 전환되고, 3D 시뮬레이션은 **수면 케어 장면(SLEEP_SCENE)** 으로 바뀝니다.  
> 남편 화면에는 늦은 시간의 소리와 조명을 줄여달라는 식의 **자연스러운 행동 제안**이 표시됩니다.”

**확인 포인트:**

- 감지 모드: **수면모드** (SLEEP_MODE)
- localStorage `thinq-mom-demo-scene` = `SLEEP_SCENE`

---

## 5. 휴양지모드 시연

**사용 발화 (메인):**

> “바다 보면서 쉬고 싶어”

**예비 발화:**

- “여행 가고 싶다, 너무 답답해”
- “호텔에서 휴가 보내는 기분 내고 싶어”
- “발리 가고 싶다”

**조작:** Hub 패널 → **휴양지모드** 칩 클릭

**설명 멘트:**

> “마지막으로 ‘바다 보면서 쉬고 싶어’라고 말해보겠습니다.  
> 이 발화는 단순히 여행 정보를 묻는 것이 아니라, 집 안에서 **기분 전환**이 필요한 상태로 해석됩니다.  
> ThinQ Mom은 이를 **휴양지모드**로 분류하고, 공기와 화면·조명 분위기를 휴식에 맞게 전환합니다.  
> 3D 시뮬레이션에서는 **휴양지 케어 장면(RESORT_SCENE)** 으로 전환되며,  
> 스탠바이미와 오브제컬렉션, 조명을 활용해 집 안에서도 잠시 휴양지에 온 듯한 장면을 보여주는 구조입니다.”

**확인 포인트:**

- 감지 모드: **휴양지모드** (내부 TRAVEL_MODE)
- localStorage `thinq-mom-demo-scene` = `RESORT_SCENE`

---

## 6. 아내 화면 업데이트 확인

**화면:** `/wife` → **홈** 탭

> “이제 아내 화면으로 이동하면, 방금 Hub에서 실행한 케어가 **오늘의 케어** 카드에 반영됩니다.  
> 아내는 복잡한 로그 대신, 오늘 필요한 케어를 **요약**해서 확인할 수 있습니다.”

**강조할 UI 요소:**

| 영역 | 설명 |
|------|------|
| 임신 주차 · 태명 | 온보딩에서 설정한 정보 |
| 오늘의 케어 카드 | Hub `mode_runs` 기반 최신 wife_card / mode fallback |
| 초음파 갤러리 | 성장 기록용 감성 해석 (의료 판독 아님) |
| AI 자동 다이어리 | 최근 7일 기록 기반 하루 정리 |

**보조 멘트:**

> “Hub에서 말한 한 문장이, 아내에게는 ‘오늘 이렇게 케어해드릴게요’라는 **따뜻한 한 줄**로 돌아옵니다.”

---

## 7. 남편 화면 업데이트 확인

**화면:** `/husband` → **홈** 탭

> “남편 화면은 아내의 **세부 증상**이나 **민감한 상태**를 그대로 보여주지 않습니다.  
> 대신 **오늘 할 수 있는 행동**으로 바꿔서 제안합니다.”

**모드별 예시 (화면에 맞춰 짧게):**

| 모드 | 남편 화면 예시 |
|------|----------------|
| 입덧모드 | “오늘 저녁은 향이 강하지 않은 담백한 메뉴가 좋아요.” |
| 수면모드 | “오늘 밤은 조용한 환경을 만들어주면 좋아요.” |
| 휴양지모드 | “오늘은 집에서도 기분 전환이 될 수 있게 함께 쉬는 시간을 만들어보세요.” |

**강조:**

- 감시가 아니라 **배려**
- 상태 공유가 아니라 **행동 번역**
- 아내가 몰라도 자연스럽게 도울 수 있는 제안

---

## 8. 초음파 갤러리 시연

**화면:** `/wife` → 초음파 갤러리 카드 → 업로드 (데모 이미지)

> “부수 기능으로 초음파 사진을 업로드하면, **의료 판독이 아니라** 성장 기록용 **감성 해석**을 제공합니다.  
> 현재 임신 주차를 기준으로 아기 크기를 **과일**에 비유하고, 짧은 **아기 말투** 메시지로 저장합니다.”

**주의 멘트 (필수):**

> “이 기능은 **의료 진단이 아니라**, 가족이 함께 기록을 남기기 위한 **감성 기록** 기능입니다.”

---

## 9. AI 자동 다이어리 시연

**화면:** `/wife` → AI 자동 다이어리 카드 → 생성

> “AI 자동 다이어리는 최근 **7일**의 상태 기록, Hub 대화, 가전 실행 기록을 바탕으로  
> 임산부가 직접 쓴 것 같은 **하루 기록**을 생성합니다.  
> 분석 리포트가 아니라, 하루를 **따뜻하게 정리**하는 다이어리 형태입니다.”

**강조:**

- 최근 7일 seed / `mode_runs` 반영
- 공기청정기·ThinQ ON 케어 기록 반영
- 임신 주차와 태명이 자연스럽게 포함

---

## 10. 마무리

**마무리 멘트 (택 1):**

> “ThinQ Mom은 임산부가 힘든 순간을 설명하지 않아도, 그 순간의 조건을 **집이 먼저** 바꿔주는 경험입니다.”

또는

> “ThinQ Mom은 가전 제어를 넘어, 임산부와 가족의 하루를 함께 돌보는 **ThinQ ON 기반 케어 경험**입니다.”

**질의 응답 전 한 줄 (선택):**

> “Hub 한 문장 → AI 해석 → 실제 가전 → 3D scene → 아내·남편 화면까지, 하나의 케어 흐름으로 연결됩니다.”
