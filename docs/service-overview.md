# ThinQ Mom 서비스 개요

최종 발표자료·산출물용 서비스 설명 문서입니다.  
브랜드명은 **ThinQ Mom**으로 통일합니다.

---

## 1. 서비스 한 줄 정의

**ThinQ Mom**은 임산부의 자연스러운 한마디를 AI가 해석해, 집 안 환경과 가족의 행동을 함께 바꾸는 **ThinQ ON 기반 임산부 케어 서비스**입니다.

**짧은 버전:** 말 한마디로 임산부의 하루를 돌보는 ThinQ ON 기반 케어 경험

---

## 2. 문제 정의

임산부는 일상 속에서 반복적으로 아래 문제를 겪습니다.

- **입덧, 냄새 민감, 피로, 수면 불편, 답답함**이 반복된다.
- 하지만 매번 **가전을 직접 조작**하거나 **가족에게 설명**하는 것은 부담스럽다.
- 남편은 도와주고 싶어도 **지금 무엇을 해야 할지** 모른다.
- 기존 스마트홈은 **기기 제어 중심**이라, 임산부의 **상태와 생활 맥락**을 충분히 해석하지 못한다.

**산출물용 요약 문장**

임산부의 어려움은 특정 순간에만 발생하는 것이 아니라, 냄새·피로·수면·기분 전환처럼 **일상 환경과 연결**되어 반복적으로 나타납니다. 그러나 기존 스마트홈 경험은 사용자가 직접 기기를 선택하고 명령해야 하는 구조에 머물러 있어, **임산부의 생활 언어를 케어 행동으로 전환**하는 데 한계가 있습니다.

---

## 3. 핵심 인사이트

1. 임산부는 “입덧모드 켜줘”라고 말하기보다 **“냄새 때문에 못 먹겠어”**라고 말한다.
2. 중요한 것은 **기기 제어**가 아니라 **힘든 순간의 조건을 바꾸는 것**이다.
3. 남편에게 필요한 것은 아내의 **상태 정보 전체**가 아니라 **오늘 할 수 있는 행동 제안**이다.
4. **허브**는 사용자의 말을 모드로 바꾸고, **앱**은 그 결과를 케어 정보와 기록으로 남긴다.

**핵심 문장**

> **기존 ThinQ가 가전을 움직였다면, ThinQ Mom은 임산부가 힘든 순간의 조건을 바꿉니다.**

---

## 4. 서비스 컨셉

ThinQ Mom은 임산부가 **평소처럼 말한 문장**을 AI가 해석해, **입덧모드·수면모드·휴양지모드** 같은 목적형 케어 모드로 전환하고, **공기청정기**와 **3D 시뮬레이션**, **아내/남편 화면**, **다이어리 기록**까지 연결하는 서비스입니다.

ThinQ Mom은 “공기청정기 제어 서비스”가 아니라, **임산부의 상태를 해석해 생활 환경과 가족의 행동을 함께 바꾸는 서비스**입니다. 가전은 케어를 실행하는 수단이며, 경험의 중심은 **사용자의 말과 그 순간의 조건**에 있습니다.

---

## 5. 핵심 경험 구조

| 단계 | 내용 |
|------|------|
| 1 | 임산부가 **ThinQ ON 허브**에 자연스럽게 말한다. |
| 2 | AI가 발화의 **문맥과 감정 신호**를 해석한다. |
| 3 | 적절한 **케어 모드**로 분류한다. |
| 4 | **공기청정기**와 집 안 환경을 전환한다. |
| 5 | **3D 시뮬레이션**이 모드별 장면으로 바뀐다. |
| 6 | **아내 화면(엄마품)**에 오늘의 케어 정보가 업데이트된다. |
| 7 | **남편 화면(아빠손길)**에는 세부 증상 대신 **행동 제안**이 전달된다. |
| 8 | **초음파 갤러리**와 **AI 자동 다이어리**로 하루 기록이 남는다. |

**경험 설명 예시**

ThinQ Mom의 경험은 **명령어 입력**이 아니라 **생활 언어 해석**에서 시작됩니다. 사용자가 “나 지금 입덧이 심해”라고 말하면 AI는 이를 **냄새 민감**과 **식사 부담** 신호로 해석하고, 입덧모드 실행과 함께 아내·남편 화면에 **각기 다른 케어 정보**를 전달합니다.

## 6. 주요 기능

### 1. AI 발화 해석 기반 케어 모드

- GPT 중심 **문맥 분류**
- 직접 키워드가 없어도 **의미 기반** 모드 판단
- 입덧·수면·휴양지 등 **시연 메인 모드** 지원
- API 실패 시 **키워드 fallback**으로 시연 안정성 확보

### 2. ThinQ ON 허브 실행

- **허브 이미지 중심**의 미니멀 첫 화면 (텍스트·버튼 없음)
- 이미지 탭 후 **음성/텍스트** 실행 패널
- 마이크 실패 시 **예시 발화 칩**·텍스트 입력으로 fallback

### 3. 모드별 환경 전환

| 모드 | 목적 |
|------|------|
| 입덧모드 | 냄새 부담 감소 |
| 수면모드 | 취침 환경 준비 |
| 휴양지모드 | 집 안에서 기분 전환 |

- **공기청정기** 실제 ThinQ API 제어
- **3D 시뮬레이션** scene 연동 (NAUSEA_SCENE / SLEEP_SCENE / RESORT_SCENE)

### 4. 엄마품 화면 (`/wife`)

- **오늘의 케어** 카드 (Hub `mode_runs` 연동)
- **초음파 갤러리** (성장 기록용 감성 해석)
- **AI 자동 다이어리** (최근 7일 맥락)
- **홈 / 마이페이지** 탭 구조

### 5. 아빠손길 화면 (`/husband`)

- **오늘의 추천** (행동 중심)
- **오늘의 집안일** 제안
- 아내 **상태·증상 직접 노출 없이** 행동 제안
- **홈 / 마이페이지** 탭 구조

### 부수 기능

- 초음파 사진 업로드
- 임신 주차별 **과일 비유**
- 짧은 **아기 말투** 메시지
- 최근 7일 상태·가전 로그 기반 **AI 다이어리** 생성

> 초음파 갤러리는 **의료 진단이 아닌**, 가족이 함께 남기는 **성장 기록용 감성 해석** 기능입니다.

## 7. 시연 플로우

발표자료·라이브 시연에 그대로 사용할 수 있는 단계입니다.

### 1. Hub 첫 화면

- ThinQ ON **이미지 하나만** 표시
- 허브를 **터치**해 실행 패널 오픈

### 2. 입덧모드 시연

- 발화: **“나 지금 입덧이 너무 심해”**
- 공기청정기 작동
- 3D **입덧 케어 장면** 전환
- 아내/남편 화면 업데이트

### 3. 수면모드 시연

- 발화: **“나 이제 잘 준비 할래”**
- 수면 환경 준비
- 3D **수면 장면** 전환

### 4. 휴양지모드 시연

- 발화: **“바다 보면서 쉬고 싶어”**
- **휴양지 장면** 전환
- 스탠바이미·오브제컬렉션·조명 활용 컨셉

### 5. 아내 화면 확인

- 오늘의 **케어 카드**
- **초음파 갤러리**
- **AI 자동 다이어리**

### 6. 남편 화면 확인

- 오늘의 **추천**
- 오늘의 **집안일** 제안
- 세부 증상이 아닌 **행동 제안**

### 7. 부수 기능 시연

- 초음파 사진 업로드 · **과일 비유**
- **AI 자동 다이어리** 생성

## 8. 차별점

| 구분 | 기존 스마트홈 / ThinQ | ThinQ Mom |
|------|----------------------|-----------|
| **시작점** | 기기 선택과 명령 | 임산부의 **자연스러운 한마디** |
| **해석 방식** | 명령어 기반 | **문맥·감정 신호** 기반 AI 해석 |
| **실행 결과** | 단일 가전 제어 | **환경 전환 + 가족 행동 제안 + 기록** |
| **허브 역할** | 음성 명령 수신 | **생활 언어 → 케어 모드** 변환 |
| **가족 연결** | 제한적 | 남편에게 **행동 중심** 제안 (증상 비노출) |
| **기록** | 기기 로그 중심 | **케어 카드 · 초음파 · 다이어리** |
| **3D / 몰입** | 없음 | 모드별 **scene** 연동 |

**한 줄 차별점**

기존 ThinQ가 **가전을 움직였다면**, ThinQ Mom은 임산부가 **힘든 순간의 조건**을 바꿉니다.

## 9. 기대효과

### 임산부 관점

- 힘든 순간을 **매번 설명하지 않아도** 됨
- 가전을 **직접 조작하는 부담** 감소
- 하루의 상태와 케어가 **자연스럽게 기록**됨
- 복잡한 설정 없이 **허브 한 번**으로 케어 시작

### 남편 / 가족 관점

- **무엇을 도와야 할지** 명확해짐
- 민감한 상태를 **직접 확인하지 않고도** 행동 가능
- 배려가 **자연스러운 루틴**으로 연결됨
- 감시가 아닌 **행동 번역** 중심 경험

### LG ThinQ 관점

- 단순 **가전 제어**를 넘어 **생활 케어 플랫폼**으로 확장
- **ThinQ ON 허브**의 역할 강화 (집의 케어 입구)
- **가전 · AI · 가족 경험**이 연결되는 신규 사용 맥락 제안
- 특정 라이프스테이지(임신·육아) **목적형 모드** 확장 가능성

**기대효과 한 줄**

가전 제어는 **케어**가 되고, 가족의 배려는 **행동**으로 이어집니다.
