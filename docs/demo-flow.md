# 시연 흐름

이 흐름은 Vercel에 배포된 화면을 기준으로 실제 사용하는 기능만 순서화한 것입니다.

## 준비

1. 모바일 또는 모바일 크기 브라우저에서 Vercel 배포 주소의 `/`를 엽니다.
2. 발표 노트북 또는 큰 화면에서 `/simulation-3d/index.html`을 엽니다.
3. StandbyMe 시연이 있으면 `/standby-display`를 엽니다.
4. 모바일과 3D 화면의 마이크 권한을 허용합니다.
5. 실제 전구 시연이 필요하면 발표 노트북에서 FastAPI 서버와 ngrok을 먼저 실행합니다.

## 모바일 첫 진입

1. `/`에서 온보딩을 진행합니다.
2. 임신준비중/임신중, 아내/남편, 임신 시작일 또는 주차, 아기 이름을 설정합니다.
3. 홈 탭에서 오늘 컨디션, 주차/준비 상태, 케어 제안, 기기 상태 요약을 확인합니다.

## 기록 탭 시연

1. 기록 탭에서 다이어리 캘린더를 엽니다.
2. 오늘 케어와 대화 기록을 바탕으로 AI 다이어리를 생성합니다.
3. 초음파 사진 업로드를 열고 이미지를 등록합니다.
4. 분석 결과, 성장 기록, AI 다이어리 스니펫, 주차별 성장 갤러리를 확인합니다.

## HUB 음성 시연

1. 모바일 앱 하단 가운데 HUB 버튼을 길게 누릅니다.
2. 대표 문구를 말합니다.
3. 앱이 음성을 텍스트로 바꾸고 의도를 분석합니다.
4. TTS 응답, 모바일 카드 상태, 3D 장면, StandbyMe 화면, 공기청정기/전구 상태가 함께 바뀌는지 확인합니다.

## 3D 시뮬레이터 시연

1. `/simulation-3d/index.html`에서 "하이 마더"라고 말합니다.
2. "네 말씀하세요" 응답 후 케어 명령을 말합니다.
3. 3D 공간, 조명 색상, 공기청정기 표현, TTS 응답을 확인합니다.
4. 10초 동안 추가 발화가 없으면 기본 화면으로 돌아가는지 확인합니다.

## 수동제어 시연

1. 모바일 앱의 수동제어 탭으로 이동합니다.
2. 현재 상태에 맞는 케어 모드 버튼을 누릅니다.
3. 공기청정기 토글과 거실 조명 토글을 실행합니다.
4. 기기 카드, 3D 화면, StandbyMe 화면이 같은 상태로 동기화되는지 확인합니다.

## 추천 시연 문구

### 임신준비중

| 순서 | 문구 | 기대 결과 |
| --- | --- | --- |
| 1 | 오늘 컨디션이 별로야. | 컨디션 밸런스 |
| 2 | 오늘은 푹 자고 싶어. | 수면 리듬 |
| 3 | 집에만 있으니까 너무 답답해. | 마음 환기 |
| 4 | 너무 지친다. | 휴식 준비 |
| 5 | 예쁜 곳에서 저녁 먹고 싶어. | 둘의 저녁 |

### 임신중

| 순서 | 문구 | 기대 결과 |
| --- | --- | --- |
| 1 | 냄새 때문에 너무 힘들어. | 입덧 케어 |
| 2 | 왜 이렇게 잠이 안들지. | 수면 케어 |
| 3 | 몸이 너무 무거워. | 가사 케어 |
| 4 | 시원한 바다 보고 싶어. | 바다 휴양 |
| 5 | 조용한 숲에 가고 싶어. | 숲 휴양 |
| 6 | 도시 야경 보고 싶어. | 도시 휴양 |

## 기기 제어 문구

| 문구 | 기대 결과 |
| --- | --- |
| 공기청정기 켜줘. | ThinQ 공기청정기 ON |
| 공기청정기 꺼줘. | ThinQ 공기청정기 OFF |
| 전구 켜줘. | 거실 조명 ON |
| 전구 꺼줘. | 거실 조명 OFF |
| 기본 모드로 바꿔줘. | 케어 모드와 조명 기본 대기 상태 복귀 |

## 최종 체크

- `/`, `/simulation-3d/index.html`, `/standby-display` 접근 가능
- 마이크 권한 허용
- OpenAI, Supabase, ThinQ 환경변수 설정
- 전구 시연 시 FastAPI/ngrok/Vercel 환경변수 최신화
- 모드 실행 로그가 `mode_execution_logs`에 저장되는지 확인

현재 기준: Vercel 시연 기능만 반영.
