# ThinQ Mom 발표자료용 문장

현재 Vercel 최종 시연 기능만 기준으로 정리한 발표 문구입니다.

## 한 줄 소개

> ThinQ Mom은 임신 준비중이거나 임신중인 사용자의 말 한마디를 공기, 화면, 조명과 하루 기록으로 연결하는 AI 홈케어 서비스입니다.

짧은 버전:

> 말 한마디로 상태에 맞는 집 안 케어를 시작합니다.

## 문제 정의

- 사용자는 기기 모드 이름보다 “잠이 잘 오게 해줘”처럼 일상적인 문장으로 말합니다.
- 임신 준비중과 임신중에는 필요한 생활 케어가 다릅니다.
- 아내와 남편에게 필요한 안내와 기록의 관점도 달라야 합니다.
- 기존 스마트홈은 기기를 제어하지만, 사용자의 상태와 가족 역할까지 이어서 설명하지는 못합니다.

## 핵심 컨셉

> 상태와 역할을 이해한 허브가 사용자의 말을 케어 모드로 바꾸고, 실제 공기청정기와 3D 홈, 모바일 기록을 함께 바꿉니다.

## 최종 시연 화면

1. 모바일 `/`: 상태·역할 선택, 홈, AI 다이어리, 디바이스
2. 허브 `/hub`: 음성 인식, 케어 실행, 음성 답변
3. 3D `/simulation-3d/index.html`: 모드별 집 안 장면

세 화면은 새로 열지 않고 기존 창에서 내용만 바뀝니다.

## 핵심 기능

1. **상태·역할 분리**: 임신 준비중/임신중, 아내/남편 조합별 화면과 응답
2. **생활 언어 해석**: 음성과 텍스트를 준비 또는 임신중 케어 모드로 변환
3. **실제 ThinQ 제어**: 공기청정기 전원, 자동, 터보, 수면 모드 실행
4. **3D 홈 연동**: 입덧, 수면, 가사, 바다·숲·도시 장면 전환
5. **모바일 결과 반영**: 공기청정기, 스탠바이미, 조명 상태 표시
6. **AI 다이어리**: 상태·역할·주차와 최근 케어 기록을 하루 기록으로 생성
7. **성장 기록**: 임신중 아내의 초음파 업로드, 갤러리, 주차별 과일 비교

## 시연 흐름 문장

> 모바일에서 임신중 아내와 임신 주차를 선택하면 허브와 3D가 같은 사용자를 반영합니다.

> “음식 냄새 때문에 속이 울렁거려”라고 말하면 입덧 케어가 실행되고, 공기청정기는 터보 모드로, 3D는 주방 공기 케어 장면으로 바뀝니다.

> 같은 결과가 모바일 디바이스 탭에 반영되고, 허브 대화와 실행 기록은 AI 다이어리의 재료가 됩니다.

> 역할을 남편으로 바꾸면 같은 케어라도 배우자를 돕는 행동 제안과 남편 관점의 기록으로 바뀝니다.

## 상태별 발화 예시

### 임신 준비중

- 아침 컨디션을 맞춰줘.
- 수면 리듬을 맞춰줘.
- 마음을 환기하고 싶어.
- 편안하게 쉬고 싶어.
- 우리 둘의 저녁을 준비해줘.

### 임신중

- 음식 냄새 때문에 속이 울렁거려.
- 잠이 잘 오게 해줘.
- 빨래와 청소를 도와줘.
- 바다를 보며 쉬고 싶어.
- 좋은 아침이야.

## 차별점

| 일반 스마트홈 | ThinQ Mom |
| --- | --- |
| 기기와 모드를 직접 선택 | 생활 문장을 상태에 맞게 해석 |
| 하나의 사용자 응답 | 임신 상태와 아내·남편 역할을 분리 |
| 단일 기기 제어 | 실제 공기청정기, 3D, 모바일을 함께 변경 |
| 기기 로그 중심 | 대화와 케어를 AI 다이어리로 기록 |

## 마무리 문장

> ThinQ Mom은 가전 제어를 사용자의 상태에 맞는 생활 케어로 바꿉니다.

> 한 번의 말이 공기와 화면을 바꾸고, 가족의 행동과 하루 기록으로 이어집니다.
